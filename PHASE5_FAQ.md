# Phase 5 FAQ（评委高频问题）

## Q1：这和普通大模型问答有什么本质区别？

**A：** 我们输出的是“多 Agent 协作 + 辩论 + 证据链”的完整决策过程，而不是单轮文本生成。

- 过程可见：Agent 执行进度、辩论交换、降级状态都可回放。
- 结果可证：`evidence_pack` 记录结论与来源映射。
- 资产可交付：一键导出 zip，可直接用于复审与汇报。

## Q2：现场网络抖动时会不会直接演示失败？

**A：** 已实现演示保护模式。SSE 中断后会进入 90 秒状态回补窗口，自动轮询 `/status/{session_id}` 恢复展示。

## Q3：你们怎么量化“稳定性”而不是口头说稳定？

**A：** 系统提供 `demo_metrics`，包含稳定性评分、降级次数、重试次数、失败 Agent 数等指标，可直接在页面看板查看，也可导出存档。

## Q4：结论可信度如何验证？

**A：** 使用“证据覆盖率 + 来源追溯”双机制。

- `evidence_coverage_rate`：有来源支撑的结论占比。
- `evidence_pack.json`：逐条结论对应来源引用，可复核。

## Q5：如果某个 Agent 失败，是否全流程崩溃？

**A：** 不会。系统支持重试与降级策略（如 `partial`），尽量保证最终仍产出可读报告，并且会记录失败与降级证据。

## Q6：成本怎么控制？

**A：** 通过工具层统计与策略协同控制：

- `tool_metrics` 提供调用量、错误率、时延、估算成本、缓存命中率。
- 结合缓存、重试上限、降级策略，压低波动成本。

## Q7：导出的路演包有什么内容？

**A：** 默认包含以下文件：

- `report.html`（完整报告）
- `executive_summary.md`（执行摘要）
- `session_snapshot.json`
- `evidence_pack.json`
- `memory_snapshot.json`
- `demo_metrics.json`
- `tool_metrics.json`
- `workflow_timeline.json`
- `manifest.json`

## Q8：这套系统适合哪些场景？

**A：** 适合“需要可解释、可审计、可归档”的市场分析场景，例如：跨境选品评估、竞品策略复盘、投前研究材料准备。

## Q9：是否支持离线审阅？

**A：** 支持。导出的 zip 可离线分发，评审可独立查看报告、摘要、证据和指标，无需在线环境。

## Q10：如何最快复现实机演示？

**A：** 启动后端与前端后，进入首页直接点击 `评委模式 -> 3 分钟标准 -> 一键运行`，结束后点击下载按钮获取路演包。
