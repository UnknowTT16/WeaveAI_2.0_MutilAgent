# Phase 5 风险应答与现场处置手册

## 1. 使用方式

- 本文用于路演现场的“故障即应答”场景。
- 每个风险按四段处理：**症状 -> 30 秒处置 -> 备选路径 -> 赛后修复**。

## 2. 风险清单

### R1：SSE 断流或页面长时间无更新

**症状**

- 页面执行进度不再变化；
- 控制台可见连接中断；
- 评委担心系统卡死。

**30 秒处置**

- 明确说明已进入演示保护模式；
- 指向页面的恢复状态卡片（恢复中/剩余秒数/轮询次数）。

**备选路径**

- 切换 `60 秒极速` 重新演示；
- 或直接使用 `/status/{session_id}` 展示最终结果。

**赛后修复**

- 检查代理缓冲与网络链路；
- 收集当次 `workflow_timeline.json` 复盘。

### R2：模型响应慢，超过预期演示时间

**症状**

- `3 分钟标准`超过 3 分钟仍未结束。

**30 秒处置**

- 立即切换到 `60 秒极速`，强调产品具备多档位演示能力。

**备选路径**

- 使用最近一次完成会话做“结果讲解 + 导出展示”。

**赛后修复**

- 对比 `tool_metrics` 的时延分布；
- 优化重试/降级阈值与场景预设参数。

### R3：某个 Agent 失败导致评委质疑稳定性

**症状**

- 页面出现 failed/degraded 状态。

**30 秒处置**

- 主动解释系统按策略降级并继续交付，不是全链路崩溃。

**备选路径**

- 在关键指标看板展示“降级次数/重试次数/最终完成状态”。

**赛后修复**

- 复盘 `workflow_events` 与 `tool_metrics`，确认失败根因并调整策略。

### R4：结论被追问“证据在哪里”

**症状**

- 评委要求来源与可追溯证明。

**30 秒处置**

- 展示证据覆盖率指标；
- 打开导出包中的 `evidence_pack.json`，定位结论与来源映射。

**备选路径**

- 在报告内指出相关段落并回到证据文件交叉验证。

**赛后修复**

- 对低覆盖率场景补齐检索策略与模板。

### R5：下载失败或无法现场交付

**症状**

- 浏览器下载被拦截，或网络策略阻断。

**30 秒处置**

- 直接使用接口导出：`/api/v2/market-insight/export/{session_id}.zip`。

**备选路径**

- 先展示 `report.html`，会后补发 zip。

**赛后修复**

- 统一浏览器下载策略与白名单。

## 3. 现场备用命令（PowerShell）

```powershell
# 运行位置：PowerShell，项目根目录
$sid="<SESSION_ID>"

# 1) 查看状态
C:\Windows\System32\curl.exe -s "http://127.0.0.1:8000/api/v2/market-insight/status/$sid"

# 2) 导出路演包
C:\Windows\System32\curl.exe -L -o "weaveai-roadshow-$sid.zip" "http://127.0.0.1:8000/api/v2/market-insight/export/$sid.zip"
```

## 4. 现场口径（统一对外）

> 我们把不确定性前置设计成“可观测、可降级、可恢复、可导出”的工程能力，
> 因此即使现场有波动，也不会影响最终可交付价值。
